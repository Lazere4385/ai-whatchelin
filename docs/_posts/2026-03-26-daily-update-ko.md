Claude Code가 이제 자율적으로 작동한다. Anthropic은 3월 24일 Auto Mode를 출시해 에이전트가 매번 사용자에게 묻지 않고 이중 레이어 분류기를 통해 스스로 권한을 결정하도록 했다. 같은 릴리스에서 Cowork 및 Discord/Telegram Channels 통합을 위한 Computer Use 기능도 추가됐으며, 회사는 Claude Code가 연간 반복 매출(ARR) 25억 달러를 돌파했다고 확인했다. 같은 날 GitHub Copilot은 개발자 커뮤니티로부터 압도적인 반발을 받은 정책 업데이트를 발표했다.

## Claude Code: Auto Mode와 ARR 25억 달러

Auto Mode는 에이전트 코딩 워크플로우의 고질적인 마찰 지점을 제거했다 — 끊임없는 권한 요청 프롬프트. 이중 레이어 분류기 아키텍처 덕분에 Claude Code는 이제 컨텍스트를 스스로 평가하고 적절한 권한을 독립적으로 결정할 수 있어, 장시간 자동화 작업에 훨씬 더 적합해졌다. Discord와 Telegram으로의 Computer Use 확장은 해당 플랫폼을 이미 사용 중인 팀에게 새로운 자동화 접점을 열어준다. Anthropic의 매출 이정표인 ARR 25억 달러는 Auto Mode 출시에 상업적 무게를 더한다.

## Cursor: 자체 호스팅 클라우드 에이전트

Cursor는 3월 25일 자체 호스팅 클라우드 에이전트를 출시해, 코드와 도구 실행이 Cursor의 인프라가 아닌 고객 자체 네트워크 내에서 이뤄지도록 했다. 얼리 어답터는 Brex, Money Forward, Notion으로, 기업 수준의 수요를 시사한다. 규제 산업과 보안에 민감한 팀에게 온프레미스 실행 유지는 중요한 장벽을 제거해준다.

## GitHub Copilot: 데이터 학습 정책이 반발을 불러오다

GitHub은 4월 24일부터 Free, Pro, Pro+ 사용자의 상호작용 데이터가 AI 모델 학습에 사용될 것이라고 발표했다 — 사용자가 직접 옵트아웃하지 않는 한. 커뮤니티의 반응은 즉각적이고 일방적이었다: 발표에 달린 반응이 👎 59 대 🚀 3이었다. 핵심 불만은 옵트인이 아닌 옵트아웃 방식이다 — 개발자들은 명시적 동의 없이 코드를 학습에 사용하는 것이 기본 설정이 되어선 안 된다고 주장한다. 이런 종류의 신뢰 훼손은 서서히 쌓이다가 갑자기 터지는 경향이 있다.

## OpenAI: 기존 Deep Research 제거

OpenAI는 3월 26일 ChatGPT에서 기존 딥 리서치 모드를 제거했다. 사용자는 이제 현재의 딥 리서치 경험으로 안내된다. 관리적인 변화이지만 ChatGPT의 늘어나는 기능 표면을 통합하려는 OpenAI의 지속적인 노력을 반영한다.

## Claude 서비스 장애 (3월 25~26일)

Claude Opus 4.6의 대규모 장애 — 3월의 여덟 번째 주요 중단 — 로 장애 신고가 최대 4,000건에 달했고, 응답 시간이 17분까지 늘어났으며, 피크 시간대에 Claude Code가 사실상 사용 불가능했다. ARR 25억 달러를 창출하는 제품에게 안정성은 다음번 명백한 압박 지점이다.
